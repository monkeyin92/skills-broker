# Milestone v1.0: Default-Entry Trust And Growth Readiness

**Status:** ✅ SHIPPED 2026-04-23
**Phases:** 1-4
**Total Plans:** 13

## Overview

Milestone v1.0 把 `skills-broker` 从一个已经能路由与复用能力的小产品，推进成一个有明确默认入口、可复用证明回路、关键失败路径更硬、且未来宿主扩展边界更清晰的版本。

## Phases

### Phase 1: Prove The QA Hero Loop

**Goal:** 把 `website QA` 变成一个对新用户、已安装宿主、broker runtime 和 operator proof 都一致可信的第一步。  
**Depends on:** brownfield runtime truth and existing shared-home lifecycle already shipped  
**Plans:** 3 plans

Plans:

- [x] `01-01-PLAN.md` — 对齐 README、README.zh-CN、repo truth 与宿主壳，让 website QA 成为唯一 first move
- [x] `01-02-PLAN.md` — 收紧 QA precision-first 边界并锁住 miss-layer eval 证明面
- [x] `01-03-PLAN.md` — 用 verdict-first doctor 与跨宿主 proof loop 证明 same-request QA 闭环

**Details:**

- Requirements: `ENTRY-01`, `ENTRY-02`, `ROUTE-01`, `ROUTE-02`, `ROUTE-03`
- Verification: `.planning/phases/01-prove-the-qa-hero-loop/01-VERIFICATION.md` (`passed`)
- Success criteria:
  1. `README.md`、`README.zh-CN.md` 与安装后的宿主说明都把 `website QA` 放在首个 hero example，而不是与其他 family 平铺
  2. Quick Start 中存在一条可直接复制的 website QA 请求，以及明确的 `INSTALL_REQUIRED -> install -> rerun -> doctor` 操作顺序
  3. 在未安装最佳赢家时，同一句 website QA 请求返回 `INSTALL_REQUIRED`，且失败原因对 operator 可解释
  4. 安装后重跑同一句请求得到 `HANDOFF_READY`，不需要更换措辞或走新路径
  5. Claude Code 与 Codex 都能复用同一条已验证 QA lane，并保留可读的跨宿主 proof / reuse evidence

### Phase 2: Generalize Family Proofs

**Goal:** 让 proof family 与 semantic metadata 支撑 `web_content_to_markdown` 等已存在 family，同时保持 `website QA` hero lane 的产品语义不退化。  
**Depends on:** Phase 1 proof loop and existing deterministic routing model  
**Plans:** 3 plans

Plans:

- [x] `02-01-PLAN.md` — 收口 canonical family proof surface，同时保留 `websiteQaLoop` alias 与 QA-first operator truth
- [x] `02-02-PLAN.md` — 锁定 proven family metadata contract 与 web markdown-only 的 fail-closed semantic boundary
- [x] `02-03-PLAN.md` — 用 integration proof 和多表面 wording 发布 “one hero lane, two proven loops”

**Details:**

- Requirements: `ENTRY-03`, `ROUTE-04`, `GROW-01`
- Verification: `.planning/phases/02-generalize-family-proofs/02-VERIFICATION.md` (`passed`)
- Success criteria:
  1. `skills-broker doctor` 能以 family-specific summary 展示 `website QA` 与 `web_content_to_markdown` 的 install / verify / reuse truth
  2. 明确的网页转 markdown 请求稳定进入 `web_content_to_markdown` family，而不是落入 QA 或 generic unsupported
  3. semantic metadata / proof family 结构能支持后续 family 扩展，但不重开 query-native 或 package-vs-leaf 已完成迁移
  4. website QA 的默认入口语义、negative boundary 和 operator wording 在泛化后保持稳定

**Depends on:** Phase 1 and Phase 2 preserving current product behavior  
**Plans:** 4 plans

- [x] `03-01-PLAN.md` — Derive runner paths at runtime and prove awkward install-path safety
- [x] `03-02-PLAN.md` — Surface advisory persistence degradation without silent greens
- [x] `03-03-PLAN.md` — Close skipped shared-home rollback/manual-recovery branches fail-closed
- [x] `03-04-PLAN.md` — Add deterministic contributor verification preflight and repair path

**Details:**

- Requirements: `HARD-01`, `HARD-02`, `HARD-04`
- Verification: `.planning/phases/03-harden-runtime-and-verification/03-VERIFICATION.md` (`passed`)
- Success criteria:
  1. 生成的宿主 runner 对 awkward install path 有明确的转义/规避方案，并有针对性测试
  2. shared-home repair、rollback、proof rail unreadable、advisory persistence failure 等路径要么被测试锁住，要么以结构化 degraded outcome 暴露给 operator
  3. 当前 skipped 的高风险 rollback / recovery 路径被关闭或以明确技术债形式记录并附带防回归保护
  4. 本地验证链路提供确定性的依赖健康修复路径，避免贡献者因为 Rollup/Vitest optional dependency 状态不明而卡死

## Milestone Summary

**Decimal Phases:**

- 无

**Key Decisions:**

- 把 `website QA` 固定为当前版本唯一 hero lane，直到 operator-visible proof 足够硬。
- 把 `familyProofs` 设为 canonical doctor surface，同时保留 `websiteQaLoop` 作为兼容 alias。
- 宿主 runner 改为从 managed manifest 读取 `brokerHome`，避免绝对路径与 awkward path 回归。
- 把 support matrix、hero lane、second proven family 与 lifecycle wording 收口到 repo-owned operator truth contract。
- 明确 future third host 必须继续保持 thin-shell、shared-home 与 proof/reuse state 三条硬边界。

**Issues Resolved:**

- `website QA` 从 docs 文案入口推进成 host shell、doctor、routing 与 cross-host reuse 一致的证明闭环。
- `web_content_to_markdown` 获得稳定的 second-proven-family contract，不再与 hero lane 语义冲突。
- shared-home repair / rollback / advisory persistence / awkward-path runner 等高 blast-radius 风险路径被 fail-closed 或结构化 degraded outcome 锁住。
- README / README.zh-CN / TODOS / STATUS 的 operator-facing truth 不再依赖人工同步。

**Issues Deferred:**

- OpenCode 第三个 thin host shell 仍 deferred，只留下明确 readiness contract。
- 更多 proven families 与 broker-owned workflows 仍留待下一个 milestone。
- MCP discovery 从 seed/demo-grade 走向 registry-grade 仍未进入本 milestone。
- narrative-doc parity automation 与 coverage budget 还没有提升到 CI 级质量门禁。

**Technical Debt Incurred:**

- `REQUIREMENTS.md` 的 checkbox / traceability 状态落后于 phase 真相，需要在 audit 与归档阶段纠正。
- 当前 desktop runtime 的 `npm PATH` / optional dependency 噪音仍会影响个别 spot-check 验证体验。
